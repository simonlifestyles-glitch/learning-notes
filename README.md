## 🤔 程式語言與編譯器

### 人類、程式語言、電腦之間的關係

電腦只看得懂 **0 和 1**，但人類沒辦法直接用 0 和 1 下指令。  
所以我們發明了「程式語言」當作中間的翻譯橋樑。

```
人類（寫程式語言）→ 編譯器（翻譯） → 電腦（執行 0 和 1） → 結果
```

例如你寫：

```python
print("你好")
```

編譯器翻譯成機器碼後，電腦執行，畫面就會出現：`你好`

---

### 編譯器是什麼？

編譯器就是上面流程中那個**「翻譯員」**，負責把你寫的程式語言轉成電腦看得懂的機器碼。

翻譯方式有兩種：

| 方式 | 說明 | 比喻 | 例子 |
| --- | --- | --- | --- |
| **編譯（Compile）** | 先把整份程式翻譯好，再執行 | 先出版整本翻譯書 | C、C++、Go、Rust |
| **直譯（Interpret）** | 一行一行邊翻譯邊執行 | 現場同步口譯 | Python、JavaScript |

---

### 三者角色總整理

| 角色 | 是什麼 | 例子 |
| --- | --- | --- |
| **人類** | 用程式語言寫指令 | 你 |
| **程式語言** | 人類與電腦之間的溝通語言 | Python、JavaScript、C++ |
| **編譯器 / 直譯器** | 把程式語言翻譯成機器碼的工具 | CPython、GCC、javac |
| **電腦** | 只懂 0 和 1，照指令執行 | 你的電腦 |

---

### 各語言對應的編譯器

| 語言 | 編譯器 / 直譯器 | 說明 |
| --- | --- | --- |
| Python | **CPython** | Python 官網下載的就是它 |
| JavaScript | **V8 / SpiderMonkey** | 瀏覽器內建 |
| C / C++ | **GCC** | 最廣泛使用的免費編譯器 |
| C / C++ | **Clang** | Apple 主導，錯誤訊息清楚 |
| C / C++ | **MSVC** | Windows 平台主流 |
| Java | **javac** | Oracle 官方，JDK 內含 |
| C# | **Roslyn** | Microsoft 官方 |
| Go | **gc** | Go 官方編譯器 |
| Rust | **rustc** | Rust 唯一官方編譯器 |

## 🛠️ 什麼是整合式開發環境（IDE）？

IDE（Integrated Development Environment）是**幫助你寫程式的全套工具箱**，把編輯、執行、除錯等功能整合在一起。

```
IDE 不是編譯器，但它幫你「管理」並「呼叫」編譯器
```

### IDE 包含哪些功能？

- **程式碼編輯器** — 寫程式的地方，有顏色標示、自動補全
- **除錯工具（Debugger）** — 幫你找出程式哪裡出錯
- **執行環境** — 按一個按鈕就能跑程式
- **檔案管理** — 管理你的專案資料夾
- **整合編譯器** — 幫你呼叫正確的編譯器

### VS Code vs PyCharm 比較

| 比較項目 | VS Code | PyCharm |
| --- | --- | --- |
| **類型** | 輕量級編輯器 | 專業 Python IDE |
| **開發商** | Microsoft | JetBrains |
| **費用** | 完全免費 | 社群版免費／專業版付費 |
| **語言支援** | 幾乎所有語言（靠外掛） | 主打 Python |
| **啟動速度** | 快 | 較慢 |
| **Python 支援** | 需安裝外掛 | 原生內建，開箱即用 |
| **適合對象** | 多語言開發者 | 專注 Python 的開發者 |

> 💡 **小白建議：** 學 Python 的話，先用 **PyCharm 社群版（免費）** 最省事！  
> 有基礎之後再換 VS Code，學會自己設定環境也是一種能力。

| 領域 | 能做什麼 | 使用語言 / 工具 | 連結 |

| 🤖 AI 開發 | 聊天機器人、圖像辨識、推薦系統 | Python + TensorFlow / PyTorch | [進入](ai) |

### 共同基礎

不管選哪個領域，都會碰到這三件事：

- **變數** — 儲存資料（例如：姓名、數字）
- **迴圈** — 重複執行某件事
- **函式** — 把一組指令包起來，可以重複使用

> 💡 不需要全部都學！找一個你有興趣的領域，從那裡開始就對了。
